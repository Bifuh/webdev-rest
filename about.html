Project Overview and Tools/Frameworks Used

This project constitutes a single-page application focused on crimes in St. Paul, Minnesota. The initial phase involved constructing a RESTful server that seamlessly interacted with the St. Paul crimes database. To achieve this, we utilized JavaScript, Node.js, Express.js, and implemented straightforward SQL queries.

Next, the development shifted towards the creation of the web application itself, employing JavaScript and Vue.js. Data retrieval was facilitated through three APIs: the custom API developed earlier, the Leaflet API for mapping functionalities, and the Nominatim API for geocoding purposes. The visual aesthetics were enhanced using the Foundation CSS framework in conjunction with our tailored CSS styling to refine the presentation of HTML elements.

Six interesting findings

As we played around with our application, we noted the following:

  • The available crime data extends only until November 1st, 2023, prompting us to ponder the developments in St. Paul's crime landscape since that date.
  • The earliest available crime data is from August 14th, 2014.
  • During this period, the peak in crime occurrence was observed on May 29, 2020, with a total of 424 recorded incidents on that day.
  • The police department employs twelve different codes to categorize the exact same incident type, namely "Aggravated Assault, Domestic." We wonder what the difference is.
  • Capitol River, situated at the heart of St. Paul, registered the highest number of crimes among neighborhoods (considering only the 1000 most recent incidents in the database).
  • Summit Hill had the least number of reported crimes (considering only the 1000 most recent incidents in the database).
